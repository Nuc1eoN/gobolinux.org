GoboLinux 017.01 - Release Notes

Introduction

We are pleased to present release 017.01 of GoboLinux, the Linux distribution featuring a rethought file system structure. GoboLinux was created out of a desire to try new approaches in the Linux distribution design space. GoboLinux 017 continues this journey, with a focus on the exploration of novel ideas aiming to make the system simpler yet functional.

Along with this release, Hisham Murhammad, founder and steward of GoboLinux for the past 25 years, is officially stepping down! The project now continues under the stewardship of Philip J. Pokarowski (@nuc1eon).

Availability

GoboLinux is available as an ISO download which you can download from this site.

Overview of this release

Main changes:

  • Resolved documented Known issues and fixes since the 017 ISO release:
    • Include UnionFS-Fuse to workaround limitations with the sandbox in the live environment
    • Loopback network interface needs to be brought up at boot
    • Update the search URL used by the Scripts package
    • Ncurses had been replaced by NcursesW, but some packages' metadata still held references to the former
    • Update expired SSL certificates (thanks to Jean-Michel T.Dydak @yann-kaelig)
    • Fix wrong versioning of SQlite
  • Fixed a missing symlink (/usr/libexec → /System/Index/libexec)
  • Fixed some startup Tasks that where not marked as executables (Avahi, DHCPNetworkInterface)
  • Installer: Fix ClockMode (GMT/localtime) selection (and default to GMT)
  • Installer: Add support for eMMC storage devices (thanks to @TheBitStick)
  • Freshen has been ported to Python 3 (thanks to Sage I. Hendricks @sage-etcher) and been therefore re-added to Gobolinux (experimental state)
  • Updated some core packages, bumped kernel to v6.12.16 (special thanks to Samuel Dionne-Riel @samueldr for the updated GoboHide patch)
  • Updated major GoboLinux tools with lots of fixes (details not listed here)
  • Switched kernel, initramfs, firmware and squashfs to zstd compression
  • Updated Firefox default profile
  • Known outstanding issues:
    • In a VM environment, installing GoboLinux' bootloader onto a GPT partion fails. Please use an MBR partition. On bare metal it should work (or configure your bootloader manually).
    • Copy & Paste between VM and and host system does not work
